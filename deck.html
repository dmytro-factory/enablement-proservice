Customization Hierarchy
03 / 18
Five primitives, in order of adoption

Start simple. Graduate to enforcement.

Phase 1
AGENTS.md

The engineering handbook. Build commands, architecture, conventions. Injected into every session automatically.

Phase 2
Slash Commands

Repeatable prompts on demand. /security-review, /pr-review. Triggered manually, standardized output.

Phase 3
Skills

Dynamic handbooks that auto-invoke. Loaded only when relevant — credit-efficient vs AGENTS.md.

Phase 4
Custom Droids

Separate agent instances with own context window and model. Delegation, parallelism, adversarial review.

Phase 5
Hooks

Deterministic controls in the agent loop. Automated guardrails, validation, naming enforcement.

Rule of thumb: Keep AGENTS.md under 500 lines. Extract specialized knowledge into Skills. Convert consistently violated rules from Skills (advisory) to Hooks (enforced).

Hooks
05 / 18
Deterministic controls in the agent loop

Advisory becomes enforced.

pre-session
Before the droid starts

Validate AGENTS.md freshness. Check repo readiness score. Enforce branch naming conventions.

ex: if AGENTS.md > 90 days old → prompt to refresh
pre-tool
Before every tool call

Intercept file edits, shell commands, and git operations. Block dangerous patterns before execution.

ex: block rm -rf / or git push --force
post-tool
After every tool call

Validate output structure. Enforce naming conventions. Run linters and type checkers automatically.

ex: run eslint after any .js edit; fail if > 0 errors
session-end
When the session closes

Generate session summaries. Update metrics. Trigger follow-up workflows or notifications.

ex: write session.md + post credit usage to Slack
pre-commit
Before every commit

Droid Shield runs here. Secret detection, credential scanning, and policy validation before code leaves the machine.

ex: scan for sk-... keys; block commit if found
on-error
On tool failure

Retry with backoff. Escalate to human. Log for post-mortem. Prevent infinite loops.

ex: git clone fails → retry with HTTPS fallback
Hook
Rule of thumb: When a Skill is consistently violated, convert it to a Hook. Skills advise. Hooks enforce. The agent cannot proceed until the hook passes. docs →

Bundle skills, droids, and MCPs. Share across teams.

Plugin Bundle

Self-Discovery
Repo-Level

Drop a plugin into .factory/plugins/ or ~/.factory/plugins/. The droid discovers and loads it automatically. No central registry required.

  • Project-scoped plugins
  • User harness plugins
  • Auto-load on session start
  • No install step needed
Org Defaults
Fleet-Wide

Set org-level default plugins via hierarchical settings. Every droid in your fleet gets the same security review droid, the same linter hooks, the same MCPs.

  • Org > Folder > Project inheritance
  • Extension-only merge
  • Lock critical plugins at org level
  • Teams add their own on top
Distribution: Self-discovery for fast iteration. Org defaults for governance. Combine both: org provides the guardrails, teams provide the specialization. plugins docs → building plugins →

Modes
14 / 18
Three ways to work with the agent

Pick the mode for the job. Normal, Spec, or Mission.

Default
Normal Mode

Interactive TUI or inline chat in VS Code. Ask questions, edit files, run commands. Real-time back-and-forth. Best for exploration, quick fixes, and pairing.

  • Inline chat in VS Code
  • Full-screen TUI for long sessions
  • Immediate feedback loop
Planning
Spec Mode

Multi-session specs for large refactors. Capture decisions, constraints, and acceptance criteria up front. Review and approve the plan before execution. docs →

  • Living document, not a dead PRD
  • Progressive disclosure — detail when needed
  • Review and approve before any code is written
Execution
Mission Mode

Orchestrated workers, adversarial validators, and a plan you approve. The agent goes away and comes back with results. docs →

  • Orchestrator plans, workers implement
  • Fresh-context validators catch drift
  • Git-native, reviewable milestones
When to use what: Normal for quick questions and debugging. Spec for anything that needs a plan reviewed by the team. Mission for work you can describe and walk away from.

Eight pillars · concrete signals we score
Style & Validation
linter · type checker · formatter · pre-commit hooks · validation under 30s
Without this: 10-min CI loop on a missing semicolon. Agent fixes blind, waits 10 more, repeats.
Build System
single documented command · pinned deps · succeeds on fresh checkout · build < 5 min
Without this: three scripts named build.sh, build_prod.sh, old_build.sh — agent picks wrong one, hits cryptic env-var error.
Testing
unit + integration · runnable locally · unit suite < 2 min · flaky tests fixed or quarantined
Without this: 45-min suite that hits staging DB. Three flakes mask one real failure.
Documentation
AGENTS.md at root · setup/build/test/deploy commands · troubleshooting guide · refreshed in last 90 days
Without this: README points to a wiki updated 18 months ago. Agent finds the working command in a 3-month-old Slack thread.
Dev Environment
devcontainer or CDE · tools/deps/credentials baked in · cold start < 10 min · parity with CI & prod
Without this: Python 3.9 locally, 3.11 in CI. “Works on my machine” — except where it ships.
Code Quality
avg file < 500 lines · functions < 50 lines · cyclomatic < 15 · clear module boundaries
Without this: agent opens a 2,000-line payments.py. Every change has unknown blast radius.
Debugging & Observability
structured logging · distributed tracing · key metrics · errors carry user/request context
Without this: “Payment failed.” That’s the whole log. Agent guesses, redeploys, waits.
Security & Governance
branch protection · secret scanning · CODEOWNERS · dep vuln scanning · CI security gates
Without this: agent inlines an API key, pushes to main. Public repo, security incident.
